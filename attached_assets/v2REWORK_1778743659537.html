1. Getting Started

Got a fresh bot? Do these steps in order.

Open the admin menu

Type /admin in the bot chat. That's where everything lives.

If /admin says you are not authorized, you are not on the admin list yet. Message support to get added.

Set up your locations

/admin → Geography Menu → Add New City

Add each city you sell in. Then under each one, add districts:

Manage Districts → pick city → Add District

Customers shop in this order: city, then district, then product type, then size. If you only sell in one place, just create one city and one district. Name them anything.

Make some product types

/admin → Products Menu → Manage Product Types

A product type is a category like "Coffee", "Snacks", "Clothing". Pick an emoji for each one. The emoji shows up in the customer menus.

Upload your first product

/admin → Products Menu → Add Products

The bot will ask you for:

  1. City
  2. District
  3. Product type
  4. Size (like 5g or 1 vnt)
  5. Price in EUR
  6. The actual file or text the customer gets after paying

Each upload is one item in stock. Ten copies of the same thing? Upload ten files. Or use Bulk Add to upload many at once.

Test it

Open a chat with your own bot from a different account. Hit /start, browse, try to buy something.

Or use the shortcut from the admin menu:

/admin → User Home

That puts you in the customer view.

Write your welcome message

/admin → Communications → Welcome Message → Add New Template

Write the intro people see when they /start. Shop name, hours, anything you want them to know.

You're done. The rest of this guide is for when you need to look things up.

5. Discounts and Sales

/admin → Discounts Menu

Four different ways to lower a price. Each one has its own little badge.

Type Badge Where you set it What it is Discount Codes (typed by buyer) Discount Codes Coupons Product Discounts fire Product Discounts Auto-applied sales Reseller Pricing crown Reseller Discounts Special prices for chosen users Tier System trophy Tier System Per-tier prices and an automatic % off based on the buyer's tier

Discount Codes (coupons)

Buyers type the code at checkout. Like SAVE10 or WELCOME50.

Discount Codes → Create New Discount Code

The bot asks for:

  1. Code name (it makes it all caps automatically)
  2. Percent off (1 to 100)
  3. How many uses — leave empty for unlimited, or set a max
  4. Does it stack with sales?
  5. Yes: applies on top of an active sale
  6. No: applies to original price only (sale ignored when this code is used)

The system always picks whichever ends up cheapest for the buyer.

To delete a code: tap the trash button next to it in the list.

Product Discounts (sales)

These are auto-applied. Buyers see the fire badge next to discounted items.

Product Discounts → Add Product Discount

Pick a scope:

  • All products of a type, anywhere
  • A specific city
  • A city plus a district
  • City plus district plus type plus size (one specific item)

Then a percent off. That's it.

Reseller Pricing

For users you've marked as resellers. They see lower prices on items with the crown badge.

Two steps:

Mark a user as reseller:

Users Menu → Manage Resellers → Add Reseller

Set the special prices:

Discounts → Reseller Discounts → Add Reseller Price

Same scope choices as product discounts.

Tier System (per-tier pricing and global discount)

Customers earn tiers automatically — see section 12 for how that works. The Tier System lets you give each tier its own pricing and an automatic % off.

Discounts → Tier System

From this screen you can:

  • Edit the tier ladder — rename tiers, change thresholds, add or remove levels.
  • Switch the tier metric — count purchases or count EUR spent. Affects how every customer's tier is calculated from now on.
  • Tier Discount Rules — like Product Discounts but limited to one tier. "10% off all Coffee for VIP only", that kind of thing. Pick scope (type / city / district / size), pick the tier, set the percent.
  • Per-Tier Global Discount — one flat % off everything in the shop, applied automatically at the bottom of the price stack. Each tier has its own number, so you can set 0% for New, 5% for Regular, 10% for VIP, 15% for Legend. Buyers in tiers with a non-zero % see a banner on their home screen.
  • Reset to Defaults — wipes back to the original ladder if you've made a mess of it.

Tier discounts apply automatically — buyers don't type anything. Items affected by a tier rule show the trophy badge.

How they all work together

Behind the scenes:

  1. Start with the catalog price
  2. Apply the fire (sale) discount if any
  3. Apply the crown (reseller) discount if the buyer is a reseller
  4. Apply the trophy (tier) rule if one matches this product and the buyer's tier
  5. Apply the buyer's per-tier global % off (if their tier has one set)
  6. At checkout, apply the typed code if the buyer enters one — and the code's "stack" setting decides whether it stacks on the already-discounted price or replaces it

The button always shows what they actually pay, with the original price crossed out next to it.

10. Auto Ads

/admin → Auto Ads System

This auto-posts your shop ad into Telegram chat groups on a schedule. It uses separate Telegram user accounts (not the bot itself) to do the posting. That's how it gets around Telegram's spam checks.

Why bother

Chat groups push old messages down fast. Manual re-posting is tedious and easy to forget. Auto Ads handles it 24/7, in many groups, with safety stuff built in.

Sub-menu

Button What Manage Accounts The user accounts that do the posting Campaigns Edit existing campaigns Create Campaign Start a new one Statistics Numbers and totals

Big picture

  1. Add one or more Telegram user accounts
  2. Each one joins the chat groups where you want ads
  3. Sync groups so the system knows which groups each account is in
  4. Make a campaign — a message plus how often plus which groups
  5. The system takes turns posting, waits in cooldowns, and bumps your ad on schedule

Adding an account (4 things)

Auto Ads → Manage Accounts → Add New Account

The bot will ask for:

1 — Account name A label for you. "Worker 1", "Vilnius Account", whatever. Up to 30 letters.

2 — Phone number Real phone number with country code, like +37061234567. It has to be a number that can get Telegram codes.

3 — API ID A number from my.telegram.org. Log in there with the same phone, click "API development tools", copy the api_id.

4 — API Hash Same page on my.telegram.org. The long string of letters and numbers next to api_id.

After step 4 the account exists but isn't logged in yet.

Logging the account in

Manage Accounts → tap the account → Authenticate Account

Telegram sends a login code to that account's official Telegram app. Type it into the bot. If 2-step verification is on, you'll also be asked for the password.

Once logged in, the account is active.

Account settings

After tapping an account in the list:

Button What Authenticate / Re-Authenticate Log in (or log in again if the session expires) View Groups All groups this account is in Sync Groups Refresh the list from Telegram Set Daily Limit Max messages this account sends per day Set Proxy Optional, for hiding the account's IP Force Resume Now Skip a cooldown Delete Account Remove (no undo)

Daily Limit — what to pick

Telegram bans accounts that post too much. The daily limit is your safety cap. New accounts should stay low. Older accounts can go higher.

Joining new groups

The account itself has to be a member before it can post. The bot doesn't auto-join.

  1. Open Telegram on the worker account
  2. Find the group, hit Join
  3. In the bot: Manage Accounts → tap account → Sync Groups
  4. The new group shows up in the list — toggle it on

Making a campaign (5 things)

Auto Ads → Create Campaign

The bot asks for:

1 — Campaign name A label. "Daily Promo", "Weekend Sale". Up to 50 letters.

2 — Account Which account does the posting. Only logged-in accounts show up.

3 — How often Frequency. Every 30 minutes, 1 hour, 2 hours, etc.

4 — Groups Either: - All Groups — every group this account is in (and toggled on) - Specific Groups — pick a subset by tapping each one

5 — Message The actual ad. Text plus optional photo or video.

After step 5 the campaign goes live right away. You can pause it whenever.

Managing campaigns

Campaigns

Each one has:

Button What Pause Stop posting (campaign stays saved) Resume Start a paused one again Delete Remove for good

Each row shows: status (running, paused, failed), account, target (all groups or how many specific), frequency.

Anti-ban stuff

Built in:

  • The accounts pretend to be on real phones (Samsung, Pixel, iPhone) instead of looking like bots
  • Auto-pause when Telegram throttles, auto-resume when it's safe
  • Daily limits stop sending when reached
  • Random spacing between posts so it looks human
  • Optional proxies

Even with all this, accounts can still get banned.

Statistics

Statistics

Shows: total accounts and how many are active, total campaigns and how many are active, lifetime messages sent, anti-ban status.

11. Workers (/klad uploaders)

/admin → Workers (/klad)

Workers are people you trust to upload products into your shop without giving them the full admin menu. They get a single command — /klad — that opens a stripped-down upload screen. They can only drop products into city/district/type/size combinations that already exist, so they can't accidentally create new categories or change pricing.

Why use this

If you have someone restocking inventory in a specific city, you don't want to hand them /admin. Workers see only what they need to: pick a location and product, send the file, done. Each upload is stamped with their @username so you can audit who added what.

Adding a worker

Workers → Add Worker

Send either:

  • A numeric Telegram user ID (like 123456789)
  • An @username

If you give a username, the user must have /start'd the bot at least once so the bot knows their numeric ID.

The Workers list

Each row shows the worker's @username (or numeric ID if no username on file), how many products they've ever uploaded, how many of those are still available, and the date they were added. Two buttons per row:

Button What it does Disable / Enable Turn off /klad access without removing the worker. Their existing uploads stay in inventory. Trash Remove for good. Existing uploads still stay in inventory — you only revoke /klad access.

What a worker sees on /klad

When the worker types /klad they get a small menu:

Button What it does Upload Product Walk through pick-city → pick-district → pick-type → pick-size → send file. Only existing combinations show up. My Uploads Their own upload history with delete buttons (only for items still available). Exit Close the menu.

Workers can't change prices, can't add new categories, can't see other workers' uploads, and can't see customer data. They only see the upload tools.

Auditing

Every upload, deletion, and toggle gets logged. You can see a worker's history under their profile in Users Menu → Search User. Products uploaded by a worker show their tag (@username) on the admin product list — useful for tracing back where a bad item came from.

12. Customer Tiers

The bot tracks every buyer's activity and gives them a tier automatically. By default the ladder is:

Tier Threshold New 0 Regular 5 VIP 15 Legend 30+

The threshold means "purchase count" out of the box. You can rename tiers, change the numbers, add more levels, or remove some. Set it up under:

/admin → Discounts Menu → Tier System

Switching the metric

The ladder can rank customers by either:

  • Purchase count — every paid order counts as 1.
  • EUR spent — sum of paid order values.

Switch from Tier System → Switch Metric. Existing tier-discount rules stay attached to their tier name, but the threshold numbers themselves don't auto-convert — review them after switching.

Editing the ladder

Tier System → Edit Tier Levels

For each level you can:

  • Rename it (anything you want — emoji are fine, like "🥉 Bronze" or "👑 Legend")
  • Change its threshold
  • Set its global discount % (see below)
  • Delete it

Or use Add Tier Level to slot in a new one. The ladder is auto-sorted by threshold, so you don't have to insert it in the right spot.

Per-tier global discount

Each tier can have its own automatic % off everything in the shop. Set it under:

Tier System → Per-Tier Global Discount

Or directly when editing a tier. Set 0% to disable for that tier. The discount applies last in the price stack, on top of any sale, reseller, or scoped tier rule the buyer already qualified for.

Buyers in a tier with a non-zero % see a banner like "🎁 Legend bonus: 15% off every item!" on their home screen so they know it's working.

Where the tier shows up

  • Their home screen at the top
  • Their profile (with a progress bar to the next tier)
  • Your user search

Customers see their tier name and progress; they don't see the threshold numbers or the discount stack — just the final price.

14. If Something Goes Wrong

"/admin doesn't work"

You're not on the admin list. Message support to get added.

"Customer paid but didn't get the item"

  1. Get their Queue ID from the order screen
  2. Tools & Settings → Payment Recovery → Lookup by Queue ID
  3. Check the order, send the item manually

If the payment really did go through but the bot missed it (rare), this manual fix sorts it out.

"Discount code isn't working"

  • Spelled right? They're auto-uppercased.
  • Out of uses? Check the code list — uses-left will be 0.
  • Was it deleted?
  • Code is set to "no stack" while there's also a sale on? In that case the bot picks whichever ends up cheaper for the buyer. If the sale is already the lower price, the code looks like it didn't apply — but the buyer is paying the lowest possible price.

"Stock count looks wrong"

  • View Bot Stock shows what's currently available, not counting reservations.
  • Purchases shows what's already sold.
  • If old basket reservations are stuck, run Clear Reservations.

"Customer says invoice expired"

Invoices last 20 minutes. If they're slow, they have to start over. Their basket items stay reserved for 15 minutes, so if they're quick they can re-checkout without losing them.

"Uploaded a product with the wrong info"

You can't edit an uploaded file directly. What to do:

  1. Find it in Manage Products
  2. Delete it
  3. Upload it again correctly

For just price corrections, Bulk Edit Prices fixes many at once.

"An Auto Ads account stopped posting"

Telegram probably put it in timeout. Usually temporary.

  1. Auto Ads → Manage Accounts → look at the account
  2. If it shows a cooldown, leave it. It'll resume on its own.
  3. If the session expired, log it in again.
  4. If it got banned for good, you'll need a new SIM and new API credentials.

"Auto Ads campaign isn't posting in a new group"

  1. Make sure the worker account is actually in the group (open Telegram on that account)
  2. Manage Accounts → tap account → Sync Groups
  3. Find the new group in the list, turn it on
  4. If the campaign targets All Groups, it'll start posting next cycle. If it targets specific groups, edit the campaign and add this one.

"The bot got banned"

If you have backup tokens, switch to one. Otherwise message support — a new token has to be made through BotFather.

"A worker can't /klad — bot says they're not registered"

  1. Workers → check the list. Are they there with a green check (Enable on)?
  2. If they're disabled, tap Enable.
  3. If they're not in the list at all, you never added them — Add Worker.
  4. If you added by @username and the bot says "user hasn't started the bot yet", tell them to send /start first, then add them again.

"Tier discount % doesn't seem to apply"

  1. Discounts Menu → Tier System → check the buyer's tier has a non-zero %.
  2. The discount applies last in the price stack — if a sale or reseller price already brought the price near zero, the % off may be small.
  3. The buyer should see the "🎁 ... bonus: X% off" banner on their home screen. If they don't, their tier is set to 0% (or they're in a tier with no global discount).
  4. Tier is cached for ~60 seconds per user — fresh purchases take up to a minute to bump them across a tier boundary.

"I switched the tier metric and now everyone's tier looks wrong"

Threshold numbers don't auto-convert when you switch between purchase count and EUR spent. A "VIP at 15" makes sense when 15 means purchases, but probably doesn't when 15 means EUR. Go into Edit Tier Levels and update each threshold to a number that makes sense for the new metric.
